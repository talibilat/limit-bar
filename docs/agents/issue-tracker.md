## Conventions

- Use `gh issue create --repo talibilat/limit-bar` when a skill says to publish an issue.
- Apply the `ready-for-agent` label to fully specified implementation issues.
- Keep local markdown copies under `.scratch/` when useful for planning or offline review.
- Do not treat external pull requests as a request surface unless the user explicitly asks for that later.
